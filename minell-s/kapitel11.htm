Kapitel 11 - Sammanfattning

Som du säkert förstått har denna kurs bara tagit upp det mest grundläggande delarna i HTML-kodning. Givetvis finns det massor av andra saker som du bör känna till och söka mer kunskap om vad det gäller t ex grafisk formgivning, layout, stillmallar, gränssnitt, multimedia, läsbarhet, funktionalitet och kompabilitet.
På följande 4 sidor kommer jag att ta upp lite om några scriptspråk, programmeringsspråk, editorer, Meta-taggar, ftp, och lagar på Internet.

Planering av sidan

När du skall göra din första "seriösa" webbsida bör du först göra en ordentlig planering. Att planera, organisera och strukturera en webbsida är viktigt för det fortsatta arbetet. Gör gärna en så kallad storyboard, där du skissar upp webbsidans struktur och ungefärliga innehåll.
Andra frågor du bör besvara under planeringsfasen är:

  • Syftet - Varför gör jag denna sida?
  • Målgrupp - Vilken är min målgrupp?
  • Mål - Vad är målet med webbsidan?

Planeringsfasen är mycket större än det jag beskriver här, du kan läsa mer om det i webbdesignhäftet. Underskatta inte planeringen, du kommer själv att märka att det är en oerhörd hjälp när webbplatsen växer.

Efter HTML...

Eftersom HTML från början utformades för att kunna formatera text så blir inte alltid resultatet som önskat eftersom sidorna idag nästan enbart består av bilder, filmer o dyl. För att komma runt detta så har man skapat en del andra formateringsspråk och skriptspråk.

 

CSS-stilmallar
CSS står för Cascading Style Sheets, eller stilmall på svenska. Ett av de största problemen med HTML är dess brist på att anpassa layouter, med stilmallar kan man styra upp hur webbsidan skall se ut. När du väl har skapat dig en stillmall kan du anropa denna för alla dina HTML-dokument. Du kan skriva dina stilmallar i t.ex Anteckningar och spara med filtillägget .css. Tyvärr handskas inte webbläsarna med stillmallar på ett standardiserat sätt och det finns en del buggar.

 

XML
XML betyder eXtensible Markup Language (utökningsbart språk för märkning). I detta språk kan man utforma sina egna taggar och hela språket bygger på samma grund som HTML (SGML) gör. Med detta språk så ska det vara lättare att skapa sidor som ser likadana ut i både Netscape och Explorer. För att skapa egna märken gör man ett dokument som kallas DTD (Document Type Definition). Detta dokument ligger till grund för beskrivningen av dina märken och dess attribut.

 

XHTML
För att underlätta användandet av XML och HTML har W3C skapat en ny standard, som är en blandning av XML och HTML.

 

DHTML
För att få sina webbsidor mer dynamiska och interaktiva finns även den utökade HTML-kodningen, DHTML.

 

VRML
VRML betyder Virtual Reality Markup Language. Detta används för att skapa tredimensionella objekt. Det är ett språk för att definiera form, färg och plats på 3-dimensionella objekt. Det kan även definiera saker som ljus, dimma, material och kopplingar mellan olika objekt. VRML är inte en utökning av HTML utan ett eget språk, däremot kan VRML ingå i en HTML-sida.

 

Flash
När man känner att HTML-koden inte räcker till utan man vill ha mer av grafiken och gränssnittet, så kan Macromedias Flash vara ett alternativ. Flash är ett verktyg för att skapa interaktiva menyer och bilder, även ljud som MP3 kan användas. All information är strömmande (streaming) vilket innebär att användaren inte behöver vänta på att informationen skall laddas. För att du skall kunna se en Flash-fil krävs det en plug-in, men det vållar inga problem eftersom det finns plug-in som standard både i Explorer och Netscape idag.

Skriptspråk

Det finns många olika skriptspråk. Med dessa kan du till exempel skapa mer levande och dynamiska webbsidor än med bara ren html. Det kan vara frågan om att utöka din webbsida med en speciell funktion eller applikation. Några exempel på sådana skriptspråk följer nedan.

 

PHP
PHP, förkortningen kom ursprungligen från Personal Home Page Tools och därefter var det Pre Hypertext Processor. Den nyaste betydelsen av förkortningen (vad jag hittat) är PHP Hypertext Processor .
PHP har på kort tid blivit ett av de populäraste serverbaserade skriptspråken. Killen bakom språket heter Rasmus Lerdorf och han hade nog inte en aning om hur genomslagskraftigt språket skulle bli. Anledningarna till att språket är så populärt är att det är snabbt, stabilt och relativt enkelt att lära sig.

PHP är ett HTML-inbäddat skriptspråk som körs på serversidan, kortfattat innebär detta att webbläsaren som efterfrågar ett PHP-dokument får ett rent HTML-dokument som svar. PHP-koden tolkas alltså av servern, men skickas inte till webbläsaren utan det användaren ser som slutresultat är ett HTML-dokument.

PHP fungerar i de flesta system, som till UNIX, Linux och Microsoft. Med PHP kan man smidigt skapa databaskopplingar och göra sina webbsidor mer dynamiska. PHP i sig självt distribueras med öppen källkod. Det innebär att den som är intresserad kan se hur språket är utformat, det innebär också att det på nätet finns en massa fri kod, program och bibliotek att hämta. Vill du veta mer kan du titta in på adressen: www.php.net .

JavaScript
Ett skriptspråk framtaget av Netscape som ett komplement till HTML-koden. JavaScript skall inte förväxlas med programmeringsspråket Java även om namnet kan få det att se ut så. Anledningen till att det namnet valdes var att det utseendemässigt liknar Java och att de tillhör samma företag.

 

JScript
Ett skriptspråk framtaget av Microsoft som gjordes för att efterlikna JavaScript (ville konkurera med Netscape). De två skriptspråken fungerar väldigt lika men eftersom dom inte är helt lika så får JScript inte kallas för JavaScript.

VBScript


VBScript står för Visual Basic Scripting Edition och är en nedbantad version av Visual Basic. Skriptspråket är framtaget av Microsoft. Språket baseras som sagt på Visual Basic och är ytterligare ett försök av Microsoft att skapa ett språk som kan konkurrera med JavaScript.

 

Perl
Perl, står för Practical Extraction and Reporting Language och togs fram för UNIX datorer. Perl används bland annat för att ta emot formulärinmatningar i e-post. Det krävs dock en "tolk" för detta skriptspråk.

 

Manus

När vi ska programmera för webben så måste vi först bestämma vart de olika kommandona ska utföras. På servern eller på klienten (användarens dator). Om man vill att kommandona ska utföras på servern så kan man använda sig av flera olika tekniker. Dom två vanligaste är CGI och ASP.

Fördelen med att köra ett program hos användaren är att det går snabbare att utföra de olika kommandona. Detta eftersom man inte först måste skicka informationen till servern, behandla informationen med serverprogrammet och sedan skicka tillbaka resultatet till användaren. Datorn kan alltså reagera direkt på vad du gör.

Nackdelen med dessa "program" är att det finns många begränsningar. Du kan exempelvis inte spara ner information på servern. Du kan bara använda de språk som webbläsaren klarar av att tolka. Ett ännu större problem är att de olika webbläsarna tolkar kommandona olika vilket gör att ett program som fungerar helt utmärkt i Netscape kanske inte fungerar alls i Microsofts Internet Explorer. Ett annat problem är att de olika webbläsarna stöder olika programspråk.

 

CGI
CGI betyder Common Gateway Interface och är en standardiserad metod för att låta ett program på en webbserver kommunicera med en webbläsare. CGI används i huvudsak om operativsystemet på webbservern är någon form av UNIX och ASP om det är Windows NT. Med hjälp av dessa tekniker kan du exempelvis göra så att olika sidor visas beroende på tid och datum eller beroende på vad användaren gjort innan.
De vanligaste språken om man väljer att göra ett program med CGI metoden är: Perl, Bash, C och C++. CGI-manus anses inte vara så säkra att använda, särskilt inte för den webbserver de ligger på.

 

ASP
ASP, betyder Active Sever Pages och är framtaget av Microsoft. ASP är inget skriptspråk i sig själv utan används antingen med JavaScript, Perl, C++ eller med VBScript. ASP exekverar (kör) sin kod på servern och sedan presenterar den färdiga informationen till klienten. Detta anses vara ett mycket säkert sätt att behandla information på, bland annat använder en del Internet-banker ASP.

Programmeringsspråk

Att programmera är ett sätt att berätta för datorn hur den ska bete sig i olika situationer. De olika sätten att styra utförandet av olika funktioner kallas för programmeringsspråk eller skriptspråk.
Den stora skillnaden mellan ett formateringsspråk som HTML och ett programmeringsspråk som exempelvis C++ är att programmeringsspråket kan utföra olika saker beroende på vad användaren hittar på, medans ett formateringsspråk bara sätter upp regler för hur någonting ska se ut.

Java
Ett programmeringsspråk framtaget av Sun Microsystems. Java skiljer sig från de andra programmeringsspråken för att man kompilerar en del av programmen innan dom hamnar hos användaren. När programmet sedan körs används en typ av tolk för att översätta dom sista delarna. Anledningen till detta är att programmen ska kunna köras från vilken dator som helst som har denna tolken (plattformsoberoende). En sådan tolk finns med i både Netscape och Internet Explorer.

 

C och C++
Java är baserat på C++, som tillhör de populäraste programmeringsspråket för avancerade tillämpningar. Dessa språk är väl strukturerade och bygger på att all kod skrivs som en eller flera funktioner. Även här krävs det en kompilator.

 

Editorer

HTML-editorer är program som lägger in och formaterar HTML-taggar i den text och de bilder man vill ha med på sin webbsida. En del klarar också av att infoga JavaScript, Javaapplets, DHTML, tabeller och ramar.

En editor underlättar arbetet med webbsidor avsevärt och sparar mycket tid om du arbetar med en stor produktion. Faktum är att du slipper skriva in HTML-koden själv och i stället kan koncentrera dig på innehållet på sidan. Men för att få ut så mycket som möjligt av editorn bör du kunna HTML så att du kan gå in i HTML-koden och ändra och precisera så att du får sidan exakt som du vill. En editor är till för att underlätta ditt jobb, inte ersätta det.

Det finns en rad olika HTML-editorer. Vissa är gratisprogram som kan laddas hem från Internet medan andra finns att köpa. Det finns också två olika typer av editorer:

  • Text-editorer - Hjälper dig att skriva taggarna. Ofta en katalog med massor av taggar och attribut. Ett exempel på en texteditor är gratisprogrammet Note tab light .

  • WYSIWYG-editorer (What you see is what you get)- Här jobbar du i en grafisk miljö och behöver inte titta på HTML-koden, vilket har både för och nackdelar.


I webbläsarna Microsoft Explorer och Netscape finns också HTML-editorer som du kan bygga dina sidor i. Nackdelen med många av dessa är att de färdiga resultaten ser olika ut i olika webbläsare eftersom de ofta skriver mycket "skräp-kod".

 

Dreamweaver
Macromedias HTML-editor, Dreamweaver, är en av de ledande editorerna som dessutom går att komma över till ett hyfsat pris. Den fungerar som ett desktop publishing-program där du klistrar in bilder och skriver text där du vill ha dem och visar sidan som den skulle se ut i en webbläsare. Texthanteringen i Dreamweaver är mycket lik den i de flesta moderna ordbehandlingsprogrammen. Det finns också möjlighet att göra filöverföring direkt i programmet.
Denna kurs är kodad i Dreamweaver MX, men eftersom jag har många exempel-koder har också jag varit en hel del i HTML-fönstret och kodat om.

 

Frontpage
Är Microsofts webbredigeringsprogram som finns i en del olika versioner. Den nyaste - Frontpage 2000 är nästan helt omgjord från tidigare versioner och mycket bättre. I tidigare versioner gick programmet in och ändrade HTML-kod själv, men detta problem är nu borta. Även här finns det möjlighet att göra filöverföring direkt i programmet, om jag inte misstar mig.

Meta-taggar

De flesta som söker information på nätet använder sökmotorer som t. ex Lycos eller Altavista. Om du vill att många användare skall hitta till din webbsida bör du anmäla den till olika sökmotorer. Hur du går till väga får du läsa om på sökmotorernas supportsidor.

När din sida hittas av en söktjänst så presenteras den under det namn du har valt som titelnamn, <TITLE> . Titelnamnet fungerar som rubrik till din sida, därför är det viktigt att du väljer ett passande namn.

För att sökmotorerna skall veta vilken information din sida innehåller och för att specifiera dina sökord använder du META-taggar. Taggen skall skrivas i index-filen och kan ha attributen NAME och CONTENT.
Meta-taggen skrivs i HTML-dokumentets huvud, alltså HEAD. En del sökmotorer visar även en beskrivande text om webbsidan. Du kan även använda Meta-taggarna för att uppdatera eller ladda en ny sida efter en given tid.
Om du är nyfiken kan du titta på källkoden på denna sida (Visamenyn-->Källa) för att se mina Meta-taggar du kan också titta i taggarkivet, så förstår du hur de skall kodas.

Publicera via FTP


Detta avsnitt tar upp lite om hur man lägger ut webbsidor på Internet. Så här långt in i kursen har allt du skapat enbart sparats ner lokalt på din hårddisk.
För att din sida skall kunna hittas på Internet måste den ligga på en webbserver. En webbserver är en dator som är uppkopplad mot Internet dygnet runt. Förutom detta är datorn utrustad med speciella program som gör att den kan skicka HTML-dokument, bilder och annat som hör till en webbsida till "surfaren".
Det som i grova drag sker när du surfar är att din dator skickar en förfrågan till en webbserver som sedan skickar en kopia på webbsidan tillbaka till din dator.

Det finns ganska många webbservrar där du helt gratis kan få lite utrymme för att lägga din webbsida på se länklista . Det brukar gå till så att man skapar sig ett konto med användarnamn och lösenord. Exakt hur man går till väga står givetvis på sidan. När du ordnat ett webbkonto ska du kopiera över alla filer som utgör din webbsida till webbservern.

FTP
FTP står för File Transfer Protocol . FTP är ett protokoll för att skicka filer över Internet. Enkelt sett skulle man kunna säga att FTP är ett slags språk som din dator använder för att prata med en server och skicka filer till den.

På din dator behöver du ett program/en klient som använder FTP för att kunna skicka filer. I Windows brukar man använda programmet WS-FTP och på Mac används ofta Fetch. Programmen finns bland annat att hämta på www.download.com Sök efter WSFTP eller CUTE FTP. Välj sedan någon av FTP-programmen för nedladdning. Följ instruktionerna för nedladdningen och instruktionerna för att fylla i de obligatoriska fälten korrekt.

För att använda ett FTP-program krävs det att du är uppkopplad mot Internet och att du sedan valt vilken server du ska skicka filerna till.

De fyra obligatoriska uppgifterna som krävs vid filöverföring är:

  1. FTP-adressen till servern - Om webb-adressen (www) till din sida är www.pingvin.com så brukar FTP-adressen vara ftp.pingvin.com, men många gånger går det också bra att använda webb-adressen.
  2. Ditt användarnamn - Det namn du fått av den som tillhandahåller servern.
  3. Ditt lösenord - Tillsammans med ditt användarnamn så gör lösenordet att du ensam har tillträde till din webbplats genom FTP. Med andra ord så är du den enda som kan ändra i din hemsida.
  4. Tilldelad katalog på server - Om du bara har fått tillgång till en katalog och inte en hel domän på servern så måste du specificera detta när du kopplar upp dig mot servern. Till exempel kan du ha fått en webb-adress som är www.algonet.se/~carina. Carina är din katalog som bara du har rätt att ändra i.

Genom ditt FTP-program så ser du katalogerna i listor på samma sätt som på din egen dator. Det här fönstret får du upp på skärmen när du startar WS-FTP och alla uppgifter är korrekt ifyllda. Sedan är det bara att börja skicka filer till ditt hemsideskonto. Markera filerna till vänster och klicka på pilen som pekar åt höger mellan fönstren. Bli inte rädd när det "plingar" till i din dator - det innebär att filen/filerna du markerat har överförts korrekt. I det breda gråa fältet nedanför visas information om kommunikationen mellan servern och din dator.

Vanliga felmeddelanden

  • Om det inte visas några filer i det högra fältet så betyder det att din uppkoppling mot servern har misslyckats. Detta beror i sällsynta fall på att servern just då har kraschat. Det vanligaste felet är dock att användaren skrivit in fel FTP-adress, användarnamn eller lösenord.
  • Om du får meddelandet Can´t get host entry i det gråa breda fältet så har du skrivit in fel adress till servern.
  • Om du får meddelandet Login Incorrect så betyder det att antingen lösenordet eller användarnamnet är fel.

Den sida som du vill ha som din förstasida måste du döpa till "index.html" . (med små bokstäver!) Då vet webbläsarna att det är din förstasida. Om det inte finns en index.html i din katalog så kommer bara en lista över filerna som finns där att visas i stället när någon surfar in.

Lagar på nätet

När man pratar om Internet, namns det ofta som ett fritt media. Faktum är att det gäller hårda regler på Internet, precis som i alla andra media-produktioner.

Upphovsrätt
Den person eller företag som skapar något, t ex en bok eller ett konstverk eller en programvara, kallas för verkets upphovsman. Det är alltid upphovsmannen som får bestämma hur verket får användas och spridas. Denne person eller företag har alltså upphovsrätt enligt upphovsrättslagen. Beroende på vad som skapas så gäller upphovsrätten olika länge.

Här inräknas t ex varumärken, programmeringskod, foton, musik, texter med mera. För att kunna använda sig av ovanstående måste man först skaffa sig tillstånd från upphovsman annars bryter man mot lagen.

 

Verk på Internet
När det gäller grafik på Internet innebär upphovsrätten att det endast är tillåtet att återanvända bilder som finns utlagda på Internet om det uttryckligen står att bilderna får användas fritt. Undantaget är enkla grafiska element som inte uppvisar någon originalitet.
För ClipArt-bilder som medföljer ett bildbibliotek, gäller det att noga läsa informationen som medföljer angående återanvändning. Musik, film och texter får inte heller användas utan upphovsmannens och utgivarens tillstånd. Även typsnitt är skyddade av upphovsrätten.

PUL - Personuppgiftslagen
1998 antog riksdagen en ny lag, Personuppgiftslagen (PUL), som ersatte datalagen. Den nya lagen är anpassad till EU.
PUL är till för att skydda den enskilda personens integritet. Detta innebär att man inte får använda sig av personers namn eller personnummer utan att fråga först.
Denna lag omfattar all automatiserad behandling av personuppgifter och dessutom manuell behandling. I lagen anges också när behandling av personuppgifter är tillåten, för känsliga uppgifter finns särskilda, stränga regler. PUL har utsats för en hel del kritik pga. att lagen inte är anpassad till webben.

 

Etik och moral på Internet
Ordet etik kan definieras som " läran om vad som är rätt och orätt" . När man pratar med någon om etik och etiska frågor skall man komma ihåg att det inte finns några entydiga svar eller lösningar.
Det som känns etiskt riktigt för mig kanske inte alls stämmer med den jag pratar med. Det finns etiska regler för medier som TV, radio och press. Dessa regler är till för att skydda enskilda mot lidande genom obefogad publicering.
Men finns det etik och moral på nätet? Förbjudet och oetiskt material som kontrolleras i traditionella medier, kan spridas okontrollerat över hela världen inom loppet av några sekunder. I traditionella medier kan man oftast peka ut en ansvarig om etiska felaktigheter begåtts, medan detta är svårt på Internet.

På Internet gäller det att den enskilda användaren tar ställning till vad som är etiskt försvarbart. Eftersom Internet är ett globalt medium kan det vara svårt att skapa gemensamma etiska regler. En grundregel för vad som är lämpligt att publicera på Internet, borde vara sådant som är lagligt i andra medier. Det vill säga sådant som inte anses kränkande för annan person.

 

Netiquette - netikett
Det försvenskade ordet, kunna netikett, skulle kunna förklaras som en sammandragning av nätverksetikett. Det är viktigt att nätanvändare är medvetna om sitt ansvar för de nätresurser de kan nyttja. Internet är som sagt en sammanslagning av en massa nätverk och trafiken som passerar på Internet passerar flera olika nätverk innan det når mottagaren.
Varje nätverk har sina egna regler och föreskrifter. Det som är tillåtet i det ena systemet, kan vara strikt förbjudet i det andra.
Användandet av ett nätverk är ett privilegium och inte en rättighet om man på olika sätt missbrukar detta privilegium kan detta bli indraget. Missbruk kan vara:

  • Olämpligt språkbruk
  • Spridandet av massbrev och reklam
  • Olämpliga bilder
  • Spridande av filer som kan skada andras systemresurser till exempel virus
  • Annan överbelastning som kan skada andra personers arbeten
  • Rasism och hets mot folkgrupp
